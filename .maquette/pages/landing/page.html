worktree-aware file tool

waft

copies the files Git ignores.

Copy `.worktreeinclude`-selected ignored files between Git worktrees.

A tiny draft for the files Git leaves behind.

copy from main to feature
$ waft copy --from main --to feature
Scanned workspace: /path/to/repo
Using profile: claude
Matched 3 files from .worktreeinclude
Copied 3 files (2.1 KB)

Quick start

  1. 1
    Add a `.worktreeinclude` file to your repo.
  2. 2
    Choose a profile, or accept the default.
  3. 3
    Preview what would be copied.
  4. 4
    Run the copy command.
  5. 5
    Files appear in your other worktree.
$ waft copy --from main --to feature --profile claude --dry-run # Preview matched files
$ waft copy --from main --to feature Copied 3 files (2.1 KB)

.worktreeinclude format

A simple, gitignore-like file with a few extras for safety.

# Copy local dev files between worktrees
.env.local
target/.cache/
.idea/

# Don't copy secrets
!secrets.env

# Negate a previous pattern
!*.env.example

# Trailing slashes match directories
logs/
See full format reference
◇

Safety guarantees

◎
Preview first

Use `--dry-run` to see exactly what will be copied.

⌁
Path validation

All paths are validated and kept inside the repo.

⊘
Never copies tracked files

waft respects `.gitignore` and will not copy Git-tracked files.

△
Clear overwrite behavior

Existing files are preserved by default. Use `--overwrite` to replace.

Compatibility profiles

Built-in defaults tuned for common toolchains.

See all profiles

Default

claude

Optimized for Claude Code and similar AI agents. Keeps sessions, caches, and local context in sync.

.claude/.env.localHistoryCache
$ waft copy --profile claude→

Git-centric

git

Generic developer tools and Git helpers. Works across editors and CLI tools.

.gitconfig.locallazygit/tigrcgh/
$ waft copy --profile git→

Worktree-first

wt

Focus on worktree-local build and run artifacts. Useful when compiled languages leave state behind.

target/build/out/.cache/
$ waft copy --profile wt→

waft is a small Rust CLI for keeping ignored worktree state in sync.

It copies the files Git ignores, nothing more, nothing less.
